HTTP는 구조가 단순한 프로토콜. 리퀘스트는 변조가 가능하고,
클라이언트로부터 서버로 오는 모든 요청은 변조가 가능. 기대하고 있는 값과
언제나 다른값이 보내질 가능성이 있음. 쿼리나 폼 HTTP 헤더, 쿠키 등을 ㄱㅇ유해서
보내져 웹 어플리케이션에 취약성이 있을 경우 정보를 도둑맞거나 권한을 빼앗긴다.

-- 공격패턴 두가지 ( 능동적 공격 , 수동적 공격 )
-- 능동적 공격은 공격자가 직접 웹 애플리케이션에 액세스해서 공격 코드를 보내는
타입의 공격
-- 수동적 공격은 함정을 이용하여 유저에게 공격코드를 실행시키게 하는 공격.
직접 액세스 하지 않는다. ( 대표적 수동적 공격 XSS, CSRF(크로스 사이트 리퀘스트))등이있다.
-- 크로스 사이트 스크립팅 - XSS, Cross-Site Scripting
크로스 사이트 스크립팅은 취약성이 있는 웹 사이트를 방문한 사용자의 브라우저에서
부정한 HTML태그나 JAVASCRIPT를 동작 시키는 공격
-- SQL 인젝션 - SQL Injection
SQL 인젝션은 SQL문장의 구문을 파괴하는 공격법이다
